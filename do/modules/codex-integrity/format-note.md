### Codex integrity gate (codex-integrity module)

**Adversarial mode (ON by default):** the unified `codex-stop.sh` Stop hook AUTO-RUNS Codex (gpt-5.5, xhigh) on every turn that did real work to adversarially review the work/code/statements and BLOCKS the turn on anything Codex flags (it must clear with `DECISION: ALLOW`). The review GROUNDS in the actual changed files (not just the turn text) and explicitly checks bugs introduced or left unfixed, discovered frontier work, and created or preserved tech debt. By DEFAULT Codex runs `workspace-write` and EDITS the repo when it judges a fix better — its changes are surfaced in the Stop output and an audit log under `.claude/state/codex-integrity/` for you to verify before stopping; set `ASK_CODEX_ALLOW_EDITS=0` for read-only/advise-only. The (secret-scrubbed) turn text is sent to an external LLM before egress (fail-closed). Fail-open: if Codex is absent / errors / times out it falls back to a `do:change-skeptic` advisory and never hard-blocks. It will not loop on itself. Toggle it with `/do:run codex on|off|status|toggle` — the flag is global (`~/.claude/.do-codex-adversarial-active`; the literal content `off` is the only thing that disables it). `/do:run codex off` reverts to the advisory reminder below.

**Advisory fallback (only when adversarial mode is OFF):** with adversarial mode turned off, the Stop hook instead EMITS A REMINDER to run a Codex review (fire the `codex` skill) on an integrity-watchlist turn — it does not run codex itself. When the `codex` CLI is absent or fails, name the in-session `do:change-skeptic` agent as the degraded fallback so an integrity-watchlist turn is never left unreviewed. `run-integrity.sh` is the helper that, when invoked, applies that fallback automatically (it detects codex absent / non-zero exit / timeout / no-DECISION and emits `SOURCE: change-skeptic` so the orchestrator dispatches the agent).
