### Codex integrity gate (codex-integrity module)

**Adversarial mode (ON by default):** the unified `codex-stop.sh` Stop hook AUTO-RUNS Codex on every turn that did real work to adversarially review the work/code/statements and BLOCKS the turn on anything Codex flags (it must clear with `DECISION: ALLOW`). The review explicitly checks bugs introduced or left unfixed, discovered frontier work that can cause undesired behavior, and created or preserved tech debt. Fail-open: if Codex is absent / errors / times out it falls back to a `do:change-skeptic` advisory and never hard-blocks. It will not loop on itself. Toggle it with `/do:run codex on|off|status|toggle` — the flag is global (`~/.claude/.do-codex-adversarial-active`; the literal content `off` is the only thing that disables it). `/do:run codex off` reverts to the advisory reminder below.

**Advisory fallback (only when adversarial mode is OFF):** with adversarial mode turned off, the Stop hook instead EMITS A REMINDER to run a Codex review (fire the `codex` skill) on an integrity-watchlist turn — it does not run codex itself. When the `codex` CLI is absent or fails, name the in-session `do:change-skeptic` agent as the degraded fallback so an integrity-watchlist turn is never left unreviewed. `run-integrity.sh` is the helper that, when invoked, applies that fallback automatically (it detects codex absent / non-zero exit / timeout / no-DECISION and emits `SOURCE: change-skeptic` so the orchestrator dispatches the agent).
