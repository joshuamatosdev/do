# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

Older versions are not supported. Update to the latest `0.1.x` release before reporting.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Use [GitHub Security Advisories](https://github.com/joshuamatosdev/do/security/advisories/new)
("Report a vulnerability" on the Security tab of this repository). This keeps the details
private until a fix is released.

### What to include

- A clear description of the vulnerability and its potential impact.
- Steps to reproduce, or a minimal proof-of-concept (local / non-destructive only).
- The version of `do` you tested against (`/do:run status` prints it).
- Any relevant environment details (OS, Node version, shell).

### What to expect

- **Acknowledgment** within 5 business days.
- **Status update** (triaging, confirmed, or declined) within 10 business days.
- If confirmed: a coordinated disclosure timeline agreed with you before any public fix lands.
- Credit in the release notes if you want it.

Reports that involve active scanning of systems you do not own, social engineering, or
denial-of-service will be declined without acknowledgment.
